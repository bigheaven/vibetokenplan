# vibetokenplan.com 产品规划文档

> 版本：v0.1  
> 状态：MVP阶段（已注册域名，未上线）  
> 定位：AI Coding 工具决策与导购平台

---

# 1. 项目背景

随着 AI coding 工具快速发展（如 Copilot、Cursor、Claude Code 等），开发者面临的问题不再是“有没有工具”，而是：

- 哪个工具更适合我？
- 哪个性价比更高？
- 哪个场景该用哪个？
- token / 订阅成本是否合理？

当前市场信息分散，缺乏统一对比和决策参考。

---

# 2. 项目目标

## 2.1 核心目标

打造一个帮助开发者做 AI coding 工具选择决策的网站：

> 帮助用户“选对工具 + 少花钱 + 提高效率”

---

## 2.2 长期目标

- 成为 AI coding 工具的“决策入口”
- 建立工具对比与成本分析标准
- 构建开发者工具推荐体系
- 实现可持续内容流量与转化

---

## 2.3 非目标（明确边界）

- 不做代码托管平台
- 不做 IDE
- 不做 AI coding 工具本身
- 不做复杂 SaaS 系统（初期）

---

# 3. 用户定位

## 3.1 核心用户

- 独立开发者
- 初创公司工程师
- AI coding 工具使用者
- 学习 AI coding 的开发者

---

## 3.2 用户痛点

- 工具太多，选择困难
- 价格不透明（token / subscription）
- 不清楚适用场景
- 试错成本高
- 多工具重复订阅

---

# 4. 产品定位

## 4.1 核心定位

> AI Coding Tool Decision Hub（决策中心）

---

## 4.2 核心价值

- 帮助用户选择 AI coding 工具
- 降低试错成本
- 提供成本与收益对比
- 提供使用场景建议

---

# 5. 网站结构设计（MVP）

## 5.1 首页（Landing Page）

目标：建立认知 + 引导点击

内容结构：

- 标题：AI coding 工具选择难题
- 副标题：帮你选对工具，减少成本
- 三个入口：
  - 工具对比
  - 成本计算
  - 推荐方案

---

## 5.2 工具对比页

功能：

- 工具横向对比
- 优缺点总结
- 适用场景
- 价格结构

初期工具：

- GitHub Copilot
- Cursor
- Claude Code

---

## 5.3 成本计算页（后期重点）

功能：

- 输入使用频率
- 计算 token / 订阅成本
- 推荐最优组合

---

## 5.4 推荐页（转化核心）

功能：

- 按场景推荐工具
- 初学者 / 重度用户分类
- 一键跳转官方订阅

---

# 6. 内容策略（核心增长引擎）

## 6.1 内容类型

### 1. 工具对比类
- Cursor vs Copilot
- Claude Code 是否值得买

### 2. 场景决策类
- 写后端用什么 AI 工具
- AI coding debug 最佳工具

### 3. 成本分析类
- 一个开发者每月 AI coding 成本
- 如何减少 token 消耗

---

## 6.2 内容目标

- 获取 Google SEO 流量
- 获取开发者社区流量
- 建立“决策权威感”

---

# 7. 商业模式（初期不强依赖）

## 7.1 Affiliate 导流

通过推荐工具获得订阅分成

---

## 7.2 信息差价值

提供工具选择建议，提高转化率

---

## 7.3 未来扩展（可选）

- 高级成本分析工具
- 会员内容（最佳 prompt / workflow）
- 企业工具选型服务

---

# 8. 技术架构（MVP）

## 8.1 推荐方案

- 前端：Next.js / Vite
- 部署：Cloudflare Pages
- 域名：vibetokenplan.com

---

## 8.2 原则

- 轻量
- 静态优先
- SEO 优先
- 不做复杂后端

---
